# 正規表現のコメント規約

正規表現リテラル(`/.../`)、`new RegExp(...)`、Go の `regexp.MustCompile(...)` /
`regexp.Compile(...)` は、ぱっと見て何にマッチするかが分かりにくい。**定義の直前に、
何にマッチするかを説明する1行コメントを必ず付ける**。

```typescript
// ✅ kebab-case(小文字英数字をハイフンで連結)にマッチ
const KEBAB_CASE = /^[a-z0-9]+(?:-[a-z0-9]+)*$/;
```

```typescript
// ❌ コメントなし
const KEBAB_CASE = /^[a-z0-9]+(?:-[a-z0-9]+)*$/;
```

## ルール

- 文字クラス・先読み/後読み・キャプチャグループなど記号が多く読み取りにくいものほど、
  コメントを省略しない。
- コメントは日本語で、**何にマッチするか**を書く(正規表現の構文自体の説明ではなく、
  意図・対象を書く)。
- コメントは式のインライン(同一行末尾)ではなく、**定義の直前の行**に書く。
- 変数名(`KEBAB_CASE` など)だけで自明な場合でも、パターンが複雑なら省略しない。
